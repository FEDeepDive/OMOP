## 💁‍♀️ 스터디 소개

- 한달에 하나씩! 정해진 기간 안에 각자의 서비스를 배포하는 스터디입니다.
- 기획 → 개발 → 배포 → 공유로 이어지는 서비스 제작 사이클을 단위로 반복 실행합니다.

<br/>

## 📖 진행 방식

- 한 달 동안 주에 1회, 2시간씩 모여 각자 하나의 서비스를 개발하고 배포합니다.
- 1시간은 모각코 방식으로 각자의 프로젝트를 개발합니다.
- 1시간은 아래 내용을 중심으로 인사이트를 공유하고 논의합니다.
    - 현재까지의 진행 상황 공유
    - 개발하면서 겪은 문제, 고민, 인사이트 공유
    - AI 활용 사례 및 생산성 향상 팁 공유
- 하나의 서비스를 마무리한 후, 공유와 회고를 통해 다음 사이클을 시작합니다.


<br/>

## 📌  규칙

- 모든 코드는 각자 개인 GitHub 저장소에 먼저 업로드합니다.
